🧠 Debugging Summary – Notes App
1️⃣ HTTPS vs HTTP Error
Problem
ERR_SSL_PROTOCOL_ERROR
Cause

Frontend was calling:

const API = "https://localhost:3000";

But Express server runs on HTTP, not HTTPS.

Fix
const API = "http://localhost:3000";
Lesson

Local Express servers usually run on HTTP unless SSL is configured.

2️⃣ Password Field Mismatch
Problem

Backend expected:

req.body.password

Frontend sent:

{ email, pass }
Fix

Frontend:

body: JSON.stringify({ email, password })
Lesson

Frontend and backend field names must match exactly.

3️⃣ Wrong Redirect Syntax
Problem
window.location("./login.html");
Fix
window.location = "./login.html";
Lesson

window.location is a property, not a function.

4️⃣ Login Sending HTML Elements Instead of Values
Problem
const email = document.getElementById("email");

This sends the input element, not the value.

Fix
const email = document.getElementById("email").value;
Lesson

Always use .value when sending form data.

5️⃣ Missing Backend Route
Problem

Frontend called:

GET /notes

Backend had:

app.get("/notes/:id")

Result:

404 Not Found
Fix

Backend:

app.get("/notes", auth, async (req,res)=>{
    const notes = await Note.find({
        userId: req.user.id
    });
    res.json(notes);
});
Lesson

Frontend API endpoints must exactly match backend routes.

6️⃣ Unexpected Token <
Problem
Unexpected token '<'
Cause

Frontend expected JSON:

await res.json()

But server returned an HTML error page.

Fix

Resolve the backend error (404 / 500).

Lesson

If you see < in JSON parsing errors, it usually means HTML response instead of JSON.

7️⃣ JWT Middleware Bug
Problem

Middleware didn't stop execution when token missing.

if(!token){
   res.status(401).json(...)
}
Fix
if(!token){
   return res.status(401).json(...)
}
Lesson

Always return after sending a response.

8️⃣ Delete Function Typo
Problem

Frontend button:

deleteNode()

Actual function:

deleteNote()
Fix
<button onclick="deleteNote('${n._id}')">
Lesson

JavaScript function names must match exactly.

9️⃣ Understanding req.user
Problem

Confusion about how backend knows the user.

Flow

User logs in → JWT created

Token stored in localStorage

Token sent in request header

authorization: token

Middleware verifies token

Middleware sets:

req.user = decoded

Routes use:

req.user.id
Lesson

JWT authentication uses middleware to attach user data to the request.

🔟 Chrome Extension Errors
Problem

Console showed:

chrome-extension://...
JobCopilot
Cause

Browser extensions.

Fix

Ignore them.

Lesson

Not all console errors come from your code.
